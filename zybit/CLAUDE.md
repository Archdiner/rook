## Claude Code notes

- Before starting any task, read `DOCTRINE.md` to understand product scope and build philosophy. Read `docs/ARCHITECTURE.md` to understand what is and isn't built.
- The end-of-session documentation obligations in `AGENTS.md` apply to every Claude session. Update the relevant docs before closing the session — not after pushing code as a separate pass.
- When in doubt about whether a change belongs in the codebase, ask: does it advance the understand → watch → identify → propose → test → learn loop? If not, it probably doesn't belong.
